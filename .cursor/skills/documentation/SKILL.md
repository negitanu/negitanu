# ライティングスタイルガイド

過去の Markdown 執筆ファイル（セキュリティ対策、トラブル対応、検証、構築、マニュアル）から抽出したスタイルルール。本プロジェクトのドキュメント執筆・校正・レビュー時に必ず適用する。

## ワークフロー

### 新規執筆

1. ユーザーにドキュメント種別を確認する（下表参照）。不明なら内容から推定し、確認する。
2. 種別に応じた骨格（見出し構成）を先に提示し、合意を得てから本文を書く。
3. Tone & Voice、Sentence Structure、Vocabulary のルールに従って執筆する。
4. Formatting 節（アラート構文、箇条書き記号、コードブロック言語指定）を最終確認する。

### 校正

Formatting 節を重点確認する:

- アラート構文の使い分けが適切か
- 箇条書きが `-` であるか（`*` は不可）
- コードブロックに正しい言語タグが付いているか
- 半角英数字と全角文字の間に半角スペースがあるか

### レビュー

- Vocabulary 節: 技術用語は英語のままか、必要な補足が括弧で添えられているか
- Sentence Structure 節: 論理構造（状況 → 原因 → 対策）が守られているか
- 種別ごとの構成テンプレートに沿っているか

## Tone & Voice

- **ます体**（です・ます）を基本とする。「削除します」「確認します」「設定します」等。
- 客観的・技術的なトーンを保つ。感情表現や主観的な形容は使わない。
- テーブル内の備考や箇条書きの短い補足では体言止めも可。

## Sentence Structure

- 一文は短〜中程度に保つ。一文に複数の情報を詰め込まない。
- 手順は**番号付きリスト**（1. 2. 3.）で記述する。
- 列挙・補足には**箇条書き**（`-` を使用、`*` は使わない）を用いる。
- トラブル対応では冒頭に **Q&A 形式の「状況」節** を置く（Q. 症状 / A. 結論）。
- 論理構造は「状況説明 → 原因 → 対策・手順」の順を基本とする。
- 接続は「そのため」「ただし」「従って」「なお」を多用。「しかし」は少なめ。

## Vocabulary & Phrasing

- 技術用語は英語のまま記載する（FortiOS, LLDP, SSL-VPN, SELinux 等）。
- 日本語の補足が必要な場合は括弧で添える：`sslvpnd（FortiGate SSL-VPN daemon）`。
- 頻出パターン：「〜のため」「〜を行います」「〜する必要があります」「〜に注意してください」。
- 漢字の開き方：標準的。「事」→「こと」、「時」→「時」のまま、「出来る」→「できる」。ひらがな過多にはしない。
- 可読性を考慮し、半角英数字と全角文字の間には半角スペースを入れる。例：`FortiOS v7.2.11 以降`。

## Formatting

### 見出し

- `#`：ドキュメントタイトル（1 ファイル 1 個）。
- `##`：大セクション（目的、状況、検証構成、手順番号付き節、まとめ 等）。
- `###`：サブセクション。手順内のステップグループ。
- セクション番号を見出しに含める場合がある：`## 1. 不要インターフェースのシャットダウン`。

### アラート構文

GitHub 形式のアラートを積極的に使う。用途の使い分けは以下の通り。

| 構文 | 用途 |
|------|------|
| `> [!TIP]` | 補足情報・知っておくと便利な情報。最も多用。 |
| `> [!WARNING]` | 設定ミスや制約に関する注意喚起。 |
| `> [!NOTE]` | 参考リンク集や背景情報。 |
| `> [!IMPORTANT]` | 手順上の重要な前提条件。 |
| `> [!CAUTION]` | 運用上のリスク・責任に関する警告。 |

### コードブロック

- CLI 出力・ログ：` ```text `
- シェルコマンド：` ```bash `
- 設定ファイル：` ```yaml `, ` ```ini `, ` ```conf ` 等、言語に合わせる。
- コードブロック内にコメント（`# 説明`、`<- 説明`）を入れて読者を誘導する。

### テーブル

- 設定パラメータ一覧、環境情報、比較表にテーブルを使う。
- 列構成は `項目 | 設定値 | 備考` が基本形。

### その他

- インラインコード（`` ` ``）はコマンド名、ファイルパス、設定値、ポート番号に使う。
- **太字**は特に強調したいキーワードに限定して使う（乱用しない）。
- 区切り線（`---`）は手順の大きなステップ間の区切りに使う。
- 脚注（`[^N]`）で外部参考リンクを示す。本文中のリンクと使い分ける。
- スクリーンショットは `![image.png](...)` で適宜挿入する。

## ドキュメント種別ごとの構成

| 種別 | 基本構成 |
|------|----------|
| トラブル対応 | `# タイトル` → 最終更新日 → `## 状況`（Q&A） → `## 原因・解決方法` → `## 考察`（任意） → `## まとめ`（任意） |
| セキュリティ対策 | `# タイトル` → 対象・時期の箇条書き → 番号付き `##` セクション → `## まとめ`（対応一覧） |
| 検証 | `# タイトル` → 最終更新日 → `## 要約` → `## 経緯` → `## 検証構成` → 設定手順 → `## 挙動確認` → `## その他` |
| 構築 | `# タイトル` → `## 構築環境`（テーブル） → `## 構築`（番号付きサブセクション） → 追加設定 |
| マニュアル | `# タイトル` → 最終更新日 → `## 注記` → 手順（番号付き `###`） → 機能紹介 |
